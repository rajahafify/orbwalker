Assetgen Integration QA Guide

Step-by-step playable QA script for launching the game, clicking through the run flow, and verifying assetgen visuals in each runtime screen.

Target: 53 / 53 integrated Asset library: 55 PNG files Godot scenes: 5 smoke checks

Scope

This guide verifies the assetgen integration feature by playing through the runtime UI. Use it to confirm the integrated assets are not merely present in manifests, but actually appear correctly while the game is running.

Pass condition: main menu, combat, shop, collection, and run summary can be reached from the running game, and each screen shows the expected assetgen art without broken images, placeholders, checkerboard artifacts, or layout-blocking visuals.

Fail condition: the game cannot reach a screen, any screen shows missing texture placeholders, the wrong background appears, an icon/sheet is visibly broken, or a runtime error appears while navigating the flow.

1. Launch The Game

Start from the Godot editor

  1. Open the project in Godot.
  2. Open res://scenes/main_menu.tscn.
  3. Press Play.
  4. Wait for the main menu to finish loading.

Pass: the game opens to the main menu with no error popup and no new red errors in the output panel.

2. Main Menu QA

Inspect the menu before clicking

  1. Confirm the main menu background fills the viewport.
  2. Confirm the title logo is readable and does not have a bad alpha box around it.
  3. Confirm the outer border/frame is visible and not stretched into the buttons.
  4. Confirm the stat panel renders cleanly.
  5. Confirm footer icons such as Profile, Achievements, and Settings are not blank or checkerboarded.

Pass: all menu art is visible, readable, and aligned; no missing texture placeholders appear.

Click Collection and return

  1. Click Collection.
  2. Verify the collection screen loads.
  3. Confirm the collection background appears behind the collection content.
  4. Check that the content text remains readable against the background and scrim.
  5. Click the back/main-menu control to return to the main menu.

Pass: Collection opens and returns cleanly, with the assetgen collection background visible.

3. Start Run QA

Start the run

  1. From the main menu, click Start Run.
  2. Wait for combat to load.
  3. Do not press Continue yet; first inspect the combat screen.

Pass: the game enters combat without scene-load errors.

Inspect combat assetgen visuals

  1. Confirm the combat background fills the screen and is not blank.
  2. Confirm the enemy panel has visible enemy portrait/stage art and the intent badge is readable.
  3. Confirm the player portrait uses the integrated hero portrait and is not a placeholder rectangle.
  4. Inspect the match board: fire, ice, earth, heart, armor, and gold orbs should be visible, cleanly clipped, and not tiny inside their cells.
  5. Inspect equipment, consumable, relic, mastery, rarity, and HUD icons where present. They should look like icons, not gray placeholders.
  6. Make one or two simple drag matches if possible, then watch VFX. Orb clear effects should appear without large opaque boxes.

Pass: combat uses the assetgen background, orb sheet, icon sheet, VFX sheet, and hero portrait without obvious runtime artifacts.

4. Advance To Shop And Summary

Normal route

  1. Play the combat normally until the fight ends.
  2. Click Continue after the victory or defeat prompt appears.
  3. If the run advances to Shop, perform the Shop QA below.
  4. If the run advances to Run Summary, perform the Run Summary QA below.

Debug-assisted route, if available

If the build shows the Enemy debug button in combat, use it to reach the later screens faster:

  1. Click Enemy debug.
  2. Click the console input at the bottom of the debug overlay.
  3. Type /fight win and press Enter.
  4. Click Continue.
  5. Repeat from combat as needed to reach Shop or later progression screens.
  6. To test defeat summary directly, type /fight lose and press Enter, then click Run Summary.

Pass: debug commands queue the transition and the Continue/Run Summary button reaches the expected next screen.

5. Shop QA

Inspect shop visuals

  1. Confirm the shop background fills the screen behind the shop UI.
  2. Confirm the merchant/header area is visible and not blank.
  3. Inspect item cards and relic card icons.
  4. Confirm prices, labels, and buttons remain readable on top of the art.
  5. Click one affordable item or relic if available; verify the icon/card changes state cleanly.
  6. Click Continue / Leave Shop.

Pass: shop art appears, cards remain readable, purchase/reroll/continue states do not reveal missing textures.

6. Run Summary QA

Reach and inspect run summary

  1. Reach Run Summary through a defeat, final victory, or the debug-assisted /fight lose route.
  2. Confirm the run summary background fills the screen behind the panel.
  3. Confirm the panel, stats, equipment list, relic list, and buttons remain readable.
  4. Click New Run and confirm it starts a new run, or click Main Menu and confirm it returns to the menu.

Pass: summary background is visible, text is readable, and both summary navigation buttons work.

7. Capture Failures

Record every visual or flow issue

For each failure, capture:

  • Screen name: Main Menu, Collection, Combat, Shop, or Run Summary.
  • Exact action that triggered it.
  • Screenshot of the broken visual.
  • Whether the output panel showed a red error or warning.
  • Expected asset path if known, for example resources/art/assetgen/backgrounds/combat_background_candidate_01.png.

8. Quick Pass Checklist

  • Pass launch opens Main Menu with assetgen menu visuals.
  • Pass Collection opens from Main Menu and shows the assetgen collection background.
  • Pass Start Run opens Combat with assetgen combat background.
  • Pass Combat board orbs render from the assetgen orb sheet with clean alpha.
  • Pass Combat player portrait and item/mastery icons are visible and not placeholders.
  • Pass Combat VFX shows no opaque boxes or checkerboard.
  • Pass Shop shows assetgen shop background and readable cards/buttons.
  • Pass Run Summary shows assetgen summary background and working New Run/Main Menu buttons.
  • Pass No new red runtime errors appear during the full route.
