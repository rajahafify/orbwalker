Orbwalker Script Size Budget

This is the documented exception register for the P1-3 roadmap guardrail. The CI budget check scans production scripts under scripts/, excluding tests and probes, and fails undocumented oversized scripts while allowing named, temporary exceptions with concrete removal paths. There are currently no documented exceptions. Future documented exceptions must be pinned to ratchet baselines: they may not grow, and any reduction must lower the baseline in the same PR.

Budgets

Script kindSoft budgetScope
Controller400 measured lines*_controller.gd production scripts under scripts/.
View500 measured lines*_view.gd production scripts under scripts/.
Other production script650 measured linesModels, services, state files, presenters, registries, developer tools, and helpers under scripts/, excluding scripts/tests/ and *_probe.gd.

Measured lines exclude blank lines and full-line comments. This matches the roadmap's controller/view size evidence and keeps formatting whitespace from changing the budget. Ratchet baselines use the same measured-line rule.

Current Exceptions

None. All production scripts under scripts/ are currently within their size budgets.

Ratchet Baselines

There are no active ratchet baselines because there are no current exceptions. If a future exception is added, it must be listed here with its current measured size as the baseline. If a file falls under its normal budget, remove the exception instead.

How To Run

Local command: godot --headless --path . --script res://tools/quality/script_size_budget.gd.

A new over-budget script must either be reduced in the same change or added here with a concrete removal path. Exceptions are debt, not permission slips.
