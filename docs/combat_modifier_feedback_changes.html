Combat Modifier Feedback Changes

This note records the combat feedback and calculation changes made for the mastery strip, item timing, and armor combo behavior.

Mastery Feedback

The mastery strip now uses the matched orb boxes as the calculation surface. When a match resolves, the relevant mastery box changes from the icon view into a centered numeric value and pops as the value changes.

  • Match values update as combo order changes, so later combo steps show their current scaled contribution.
  • Fire, Ice, and Earth keep combo scaling in the preview and result flow.
  • Armor displays its unscaled match armor value instead of inheriting damage combo scaling.

Item Modifier Timing

Equipment and relics now animate when they affect a calculation. The timing is based on the modifier type so the player can see why a value changed at the moment it contributes.

  • Orb value bonuses and combo-count effects animate during match and combo preview timing.
  • Flat damage, heal, and gold bonuses animate near the end of the resolution flow before the matching result VFX.
  • Start-turn armor animates before enemy attack replay when the turn has incoming attack context.

Armor Rule Fix

Armor matches intentionally do not use the damage combo multiplier. The resolver, preview calculation, source highlighting, and combat log formula now follow the same rule.

  • CombatStateMachine resolves Armor matches with a multiplier of 1.0.
  • CombatController previews Armor feedback without combo scaling.
  • CombatTurnLogPresenter reports armor as the matched armor total instead of a damage-multiplied formula.

Verification

The targeted combat regression coverage confirms Armor remains unscaled while Fire still responds to combo scaling and item effects.

  • Godot MCP combat test harness: CombatStateMachineTest.new().run_all() passed.
  • Preview assertion: Armor 3-match at combo 3 remains 3.
  • Preview assertion: Fire match still scales with combo and bonus sources.
