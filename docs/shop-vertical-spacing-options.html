Shop Vertical Spacing

The current shop is designed at 1080x1920, then centered inside the phone viewport. On tall phones like 1080x2400, that creates empty top and bottom bands. The fix should stop centering the fixed design vertically and let the shop layout use the full logical height.

Current behavior

Fixed 16:9-ish design centered on tall phones

The content is not actually too short; it is centered inside a taller phone. That makes the shop feel like it is floating with a big unused cap above and below.

  • Root cause: _layout_root.position = (viewport_size - scaled_size) * 0.5.
  • On 1080x2400, scaled_size.y stays 1920.
  • The leftover 480px is split into top and bottom bands.

Option 4

Scale to height and crop width

This fills the screen fast, but it is the wrong kind of fix for a UI-heavy game. It risks clipping left/right controls, card edges, popovers, and touch targets.

  • Would use max(widthScale, heightScale).
  • Looks full-screen, but becomes fragile across phones.
  • Not recommended for this shop.

Execution Plan

Implemented Checkpoint
  • Replace vertical centering in ShopView.apply_layout() with a tall-viewport aware layout frame.
  • Keep width scaling stable: calculate scale from width first, then derive logical_height = viewport_size.y / scale_factor.
  • Use extra_height = max(0, logical_height - 1920) to produce adaptive rects.
  • Pin TOP_BAR_RECT near top safe area, then place merchant/stock/relic/actions below it.
  • Pin the shared player HUD section to logical_height - hud_height instead of hardcoding y = 1428.
  • Distribute surplus cautiously: merchant art can grow first; card sizes should remain unchanged.
  • Update the shop layout probe to include logical_height, top_unused_gap, and bottom_unused_gap for 1080x1920 and 1080x2400.
  • Chosen implementation: adaptive full-height layout plus larger merchant, stock, and relic sections.
  • Stopped here after phone review. A later denser space-usage exploration was rejected, so no extra density pass is included in this checkpoint.
  • Keep combat/header sharing untouched unless a shared viewport-layout helper emerges naturally.
var scale_factor := viewport_size.x / DESIGN_SIZE.x
var logical_height := viewport_size.y / scale_factor
var extra_height := maxf(0.0, logical_height - DESIGN_SIZE.y)

_layout_root.position = Vector2(
  (viewport_size.x - DESIGN_SIZE.x * scale_factor) * 0.5,
  0.0
)
_layout_root.size = Vector2(DESIGN_SIZE.x, logical_height)
_layout_root.scale = Vector2(scale_factor, scale_factor)

var merchant_extra := minf(extra_height * 0.38, 180.0)
var hud_rect := Rect2(
  Vector2(0, logical_height - 492.0),
  Vector2(1080, 492)
)

# Then derive topbar, merchant, stock, relic,
# action row, and HUD rects from these adaptive values.
