Shop And Collection Card Readability

Documented May 25, 2026. This note records the shared asset-backed card renderer, shop reuse, top-header alignment, price badge cleanup, and validation evidence.

Implementation Summary

Shared Card Renderer

The collection and shop item cards now share scripts/ui/collection_card_renderer.gd. Both screens use the same ornate frame geometry, rarity-colored surfaces, title region, art region, copy region, and bottom badge.

Shop Stock Reuse

Shop stock offers reuse the same renderer as collection upgrade cards. The shop no longer carries a separate prototype-looking card implementation for stock items.

Header Reuse

Combat and shop now use the same shared top-header scene contract for title, gold, help, and settings controls. Gold notation is standardized to compact dollar text.

Readability Rules

  • No visible rarity tag text on cards. Rarity is communicated through the card surface color.
  • Text, art, and badge regions are dedicated and do not overlap.
  • Shop card copy is compact and readable, using phrases like +2 Attack, +1 Fire Mastery, and Choose 1 of 3 Fire rewards.
  • Consumable conversion copy is explicit, for example Convert 3 non-Ice orbs to Ice orbs.
  • Prices use compact notation such as $9. Unaffordable cards keep the price and communicate disabled state visually instead of changing the label to Need X.

Badge Cleanup

The shared price badge asset at resources/art/assetgen/runtime/collection_ui/price_badge.png was cleaned so the center reads as a proper dark-gold button instead of a bright smeared highlight. Active badges no longer add an external rectangular glow or shadow; the button state is carried by the shared badge asset itself.
