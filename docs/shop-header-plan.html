Shop Header Readability Plan

Goal: rebuild the yellow-highlighted shop header into a legible, componentized hero/header area that reads closer to the reference: strong title, obvious gold counter, simple controls, and one larger merchant message panel over the shop art.

Prioritized Work

P0: Make The Header Readable

  • Split the current top strip into clear components: header.title, header.gold_counter, and header.help_button.
  • Replace tiny mixed labels with one large title: Dungeon 1-2 Shop.
  • Make the gold counter a central framed plaque with a coin medallion and at least 48px height in design space.
  • Keep controls as round icon buttons on the right, matching the reference help/settings treatment.

P1: Reframe The Merchant Section

  • Turn the merchant area into header.merchant_stage with one primary child: speech_panel.
  • Increase the speech panel text size and line height; use short, high-contrast copy like the reference: “Well met, adventurer. New stock, fresh from the depths.”
  • Remove boss preview from the header mockup; keep only the greeting panel from the reference.
  • Let the merchant/background art carry the right side instead of adding a separate status/instruction box.

P2: Improve Visual Hierarchy

  • Use a single dark framed header band instead of many small nested panels.
  • Let the background art show through behind the merchant stage, but darken it enough for text readability.
  • Add one clear bottom rail under the merchant stage so the header ends cleanly before shop stock begins.
  • Align left and right content to shared gutters; avoid floating text blocks with unrelated widths.

P3: Implementation Safety

  • Create named rect constants for each component instead of inline Rect2 calls.
  • Expose the new header geometry through shop_layout_probe_snapshot().
  • Add layout tests for minimum font zones, component containment, button hit size, and no overlap between title, gold, and controls.
  • Keep the first implementation scoped to the yellow/header section before changing stock cards, relics, or footer.

Notes For Godot Implementation

Recommended first pass: use scenes/ui/top_header.tscn as the single shared top header in shop and combat, then pair it with a single speech-panel merchant stage in shop.
Readable minimums: title 28px or larger, gold value 26px or larger, speech body 18px or larger, and help control 48px or larger.
Help behavior: the ? button should open a modal with the shop action guidance, not render that guidance inside the header.
Reference direction: less tiny dashboard chrome, more framed fantasy storefront: large title, dominant merchant area, warm gold rails, and fewer competing text blocks.

Execution Plan

Wire the help button. Keep the ? as a real shop-help control. Pressing it opens a modal with “Shop opened. Buy, reroll, sell, or continue.” and the short buy/sell instruction copy.
Extract shared header. Keep title, gold, help, settings, chrome, and geometry in scenes/ui/top_header.tscn plus scripts/ui/top_header.gd; shop and combat must reuse that scene instead of restyling private copies.
Build the topbar once. Replace the current small mixed header labels with one large title, a framed gold counter with coin medallion, and one round help control.
Rebuild the merchant stage. Keep the shop art visible, place one readable speech plaque on the left, remove the extra status/instruction panel, and add a clean bottom rail where the header ends.
Add the help modal. Implement the modal as a centered framed panel with a close button, keyboard/backdrop dismissal, and no impact on the normal header layout.
Expose layout probes. Add the shared header and help modal rects to layout probes so tests can assert containment, minimum sizes, no overlap, and shop/combat parity.
Enforce shared ownership. Keep scripts/debug/top_header_contract_probe.gd green so combat instances scenes/ui/top_header.tscn, shop instantiates the same scene, and both probes use TopHeader.layout_snapshot_for().
Validate in small passes. Update the layout test, run the focused shop layout probe, run git diff --check, then boot res://scenes/shop.tscn headlessly and visually compare against this mockup.
