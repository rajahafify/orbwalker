Godot Android Windows Recovery

This note records the Windows reformat recovery performed for the Godot 4.6.3 project at D:\godot\matchatro. The main failures were missing Android SDK/JDK tooling, stale Godot editor paths from the previous Windows user, mismatched Godot export templates, and generated Android build assets being indexed as normal project resources.

Installed Tooling

  • Temurin JDK 17: C:\Program Files\Eclipse Adoptium\jdk-17.0.19.10-hotspot
  • Android SDK: C:\Android\Sdk
  • Android command-line tools: C:\Android\Sdk\cmdline-tools\latest
  • Android platform-tools / ADB: C:\Android\Sdk\platform-tools
  • Android SDK platforms: android-35, android-36
  • Android build-tools: 35.0.1, 36.1.0
  • Godot export templates: %APPDATA%\Godot\export_templates\4.6.3.stable
  • Git for Windows 2.54.0

Godot Editor Settings

Godot was still pointing to old paths under C:\Users\Home\scoop\.... The editor settings were updated in %APPDATA%\Godot\editor_settings-4.6.tres: 

export/android/java_sdk_path = "C:\Program Files\Eclipse Adoptium\jdk-17.0.19.10-hotspot"
export/android/android_sdk_path = "C:\Android\Sdk"
export/android/debug_keystore = "C:/Users/Raja Hafify/AppData/Roaming/Godot/keystores/debug.keystore"

Android Build Template

The project Android build template had been generated for Godot 4.6.2 while the editor was running Godot 4.6.3. The active android/build folder was refreshed from the Godot 4.6.3 android_source.zip export template.

The old backup was moved outside the project tree so Godot cannot scan it: 

D:\godot\matchatro_android_backups\build_backup_4.6.2_20260521_164355

Generated Android Assets Cleanup

Godot had indexed generated files under res://android/build/..., causing:

  • Duplicate project.godot warnings
  • Duplicate UID warnings for copied assets
  • Duplicate class_name parse errors from exported .gdc files
  • Duplicate GDAI GDExtension singleton and editor plugin registration errors

The fix was to restore D:\godot\matchatro\android\build\.gdignore, remove generated Android intermediates/assets from the active Android build folder, and rebuild Godot's generated cache files under .godot.
