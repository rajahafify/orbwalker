Codebase Quality Roadmap

A prioritized, P0–P5 plan to move the codebase from a verified 8/10 to a defensible 10/10. Every item below traces to something read directly in the tree, with concrete actions and acceptance criteria so "done" is unambiguous.

How to read this

PriorityP0 = trust/correctness blocker · P5 = long-horizon polish. Do them roughly in order; later phases assume earlier ones landed.
EffortS = <½ day · M = 1–3 days · L = ~1 week · XL = multi-week, do incrementally.
AcceptanceThe green line is the objective, testable bar. If it isn't measurable, it isn't done.
ConfidenceItems marked verify rest on one fact I couldn't confirm under a degraded tool channel; check it before acting.
The thesis. This codebase is already well-architected: real MVC separation, broad behavioral tests, typed and defensive code. The gap to 10 is not "write more features" — it's removing the two god-objects, making contracts self-enforcing instead of grep-based, and proving correctness through an enforced gate. A 10/10 codebase is one where a stranger can change anything and the system tells them immediately if they broke it.

P0

Foundational integrity

Things that are actively broken, erode trust, or mean "green" doesn't yet mean "safe." Nothing else matters until the safety net is real and the repo is clean. Target: 1 week.

P0-1 Enforce the existing tests in CI

The repo now has a single headless runner that discovers existing test suites and contract probes, plus a GitHub Actions workflow that runs it on pushes and pull requests. This remains P0 until the remote CI gate has run successfully on the branch and merge protection points at it.

Actions
  • Keep scripts/tests/run_all_tests.gd as the canonical runner; it discovers every *_test.gd and *_probe.gd file that follows the shared run_all() contract.
  • Run it via godot --headless --path . --script res://scripts/tests/run_all_tests.gd in GitHub Actions against pinned Godot 4.6.3.
  • Gate merges on the workflow, then add a real status badge to CONTRIBUTING.html once the remote owner/repo badge URL is known.
Acceptance: a PR that breaks any existing assertion is blocked by CI before merge; the runner's exit code is non-zero and the failing case name is printed.
Current evidence: local Godot 4.6.3 headless run passed 45 suites and 192 cases through scripts/tests/run_all_tests.gd.

What "10/10" means here

Change without fear

Any file can be refactored and the test gate tells you in minutes whether you broke a flow — not a user, three weeks later.

Contracts enforce themselves

Invariants are checked at runtime against real objects, never by grepping source text. Renames and reformats never cause false failures.

No god-objects

Every class has one reason to change. The biggest file is comprehensible in one sitting.

Typed at the seams

Cross-module data has a declared shape; a typo is a parse/analysis error, not a silent miss.

Clean to clone

No binaries or build output in history; a fresh clone is small and builds green immediately.

Legible to a stranger

Architecture, conventions, and the decisions behind them are documented and discoverable.

Survives shipping

Save data migrates, hot paths hold a budget, debug code can't reach release, and text can be localized.

Green means safe

CI is the source of truth: format, lint, warnings, unit, contract, flow, layout, coverage, and asset validity all gate merge.
