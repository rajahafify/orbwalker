Codebase Quality Roadmap

A prioritized, P0–P5 plan to move the codebase from a verified 8/10 to a defensible 10/10. Every item below traces to something read directly in the tree, with concrete actions and acceptance criteria so "done" is unambiguous.

How to read this

PriorityP0 = trust/correctness blocker · P5 = long-horizon polish. Do them roughly in order; later phases assume earlier ones landed.
EffortS = <½ day · M = 1–3 days · L = ~1 week · XL = multi-week, do incrementally.
AcceptanceThe green line is the objective, testable bar. If it isn't measurable, it isn't done.
ConfidenceItems marked verify rest on one fact I couldn't confirm under a degraded tool channel; check it before acting.
The thesis. This codebase is already well-architected: real MVC separation, broad behavioral tests, typed and defensive code. The gap to 10 is not "write more features" — it's removing the two god-objects, making contracts self-enforcing instead of grep-based, and proving correctness through an enforced gate. A 10/10 codebase is one where a stranger can change anything and the system tells them immediately if they broke it.

P0

Foundational integrity

Things that are actively broken, erode trust, or mean "green" doesn't yet mean "safe." Nothing else matters until the safety net is real and the repo is clean. Target: 1 week.

P0-1 Enforce the existing tests in CI

The repo now has a single headless runner that discovers existing test suites and contract probes, plus a GitHub Actions workflow that runs it on pushes and pull requests. This remains P0 until the remote CI gate has run successfully on the branch and merge protection points at it.

Actions
  • Keep scripts/tests/run_all_tests.gd as the canonical runner; it discovers every *_test.gd and *_probe.gd file that follows the shared run_all() contract.
  • Run it via godot --headless --path . --script res://scripts/tests/run_all_tests.gd in GitHub Actions against pinned Godot 4.6.3.
  • Gate merges on the workflow, then add a real status badge to CONTRIBUTING.html once the remote owner/repo badge URL is known.
Acceptance: a PR that breaks any existing assertion is blocked by CI before merge; the runner's exit code is non-zero and the failing case name is printed.
Current evidence: local Godot 4.6.3 headless run passed 75 suites and 297 cases through scripts/tests/run_all_tests.gd.

P0-2 Keep generated artifacts out of the tree and history

Orbwalker.apk existed locally as an ignored root-level export artifact and has been removed from the working tree. A repo audit shows git log --all -- Orbwalker.apk already returns nothing, so the priority is preserving that invariant and keeping build/, .godot/, tmp/, logs, Python bytecode, and Android export output out of source control.

Actions
  • Move release binaries to CI artifacts / GitHub Releases. Never commit them.
  • Keep the existing ignore rules covering APKs, build/, .godot/, tmp/, logs, and Android export output aligned with the actual export workflow.
  • Keep the lightweight tracked-artifact check in the P0-1 runner and CI workflow so accidental APK/build/cache output commits fail fast.
  • Triage tmp/capture_*.gd / tmp/*_probe.gd if any are intentionally promoted later; keep local scratch output untracked by default.
Acceptance: git log --all -- Orbwalker.apk stays empty; git status --ignored --short shows no unexpected root APK/build output; no generated output is tracked.
Current evidence: repo_hygiene_test.gd shells out to git ls-files through the shared headless runner and rejects tracked APK/AAB/APKS/IDSIG files, build/editor/log/tmp/android export folders, and Python bytecode/cache output. The previously tracked tools/asset_tools/__pycache__/build_assetgen_runtime_assets.cpython-312.pyc was removed, and .gitignore now covers Python cache artifacts.

  • Wire the probe into the P0-1 runner so it can never silently rot again.

Self-enforcing contracts & type safety

Make invariants impossible to break silently. Today some contracts are enforced by grepping source text — which is exactly why a clean refactor broke one. Replace fragile string-contracts with runtime ones, and restore type safety at the boundaries the binding-dictionary erased. Target: 1–2 weeks.

P2-1 Typed accessor over the node binding dictionary

ROOT_NODE_BINDINGS + _build_root_nodes() is a clean win for de-duplication, and now has an explicit ROOT_NODE_TYPES contract so the old per-field as Label / as ProgressBar safety is restored at the scene boundary. Keep this contract current whenever a root binding changes.

Actions
  • Maintain the expected type for each binding in ROOT_NODE_TYPES.
  • Have _build_root_nodes() verify is-type on resolve and push_error on mismatch.
  • Keep combat_scene_contract_test.gd asserting type, not just presence.
Acceptance: changing a node's class in combat.tscn to the wrong type fails combat_scene_contract_test in CI with a precise message.
Current evidence: combat_scene_contract_test.gd now includes combat_scene_script_is_attached and combat_root_node_bindings_have_expected_types, and the aggregate runner passes both under Godot 4.6.3.

P2-2 Extract the tutorial system out of combat

The tutorial used to be a set of TUTORIAL_* constants and scripted swap-paths living inside combat_controller.gd, while scenes/tutorial.tscn was an empty extends Control stub. The tutorial logic is real and routes through start_tutorial_run() into combat, so it needs a cohesive owner rather than another branch inside the combat controller.

Actions

What "10/10" means here

Change without fear

Any file can be refactored and the test gate tells you in minutes whether you broke a flow — not a user, three weeks later.

Contracts enforce themselves

Invariants are checked at runtime against real objects, never by grepping source text. Renames and reformats never cause false failures.

No god-objects

Every class has one reason to change. The biggest file is comprehensible in one sitting.

Typed at the seams

Cross-module data has a declared shape; a typo is a parse/analysis error, not a silent miss.

Clean to clone

No binaries or build output in history; a fresh clone is small and builds green immediately.

Legible to a stranger

Architecture, conventions, and the decisions behind them are documented and discoverable.

Survives shipping

Save data migrates, hot paths hold a budget, debug code can't reach release, and text can be localized.

Green means safe

CI is the source of truth: format, lint, warnings, unit, contract, flow, layout, coverage, and asset validity all gate merge.

Sequencing in one line: P0 makes the net real and the repo clean → P1 removes the structural risk that the net now lets you touch safely → P2 makes contracts unbreakable → P3 deepens the net to flows → P4 automates the cheap catches and documents the why → P5 hardens for the long haul. Do P0 first and in full; it's what makes everything after it safe to attempt.
