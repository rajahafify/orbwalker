Matchatro Codebase Conventions

Architecture Map And Decisions

Use docs/architecture_decisions.html for the live contributor-facing module map, MVC command flow, and lightweight ADRs behind scene binding, signal ownership, runtime contracts, asset fallback resolution, and combat view/controller boundaries.

  • docs/system_architecture.html is the broad system architecture baseline.
  • docs/codebase_quality_roadmap.html tracks quality gaps, acceptance criteria, and current evidence.

Scene Signal Ownership

Scene-owned controls should declare their persistent signal connections in the .tscn file. Runtime-created nodes should connect their signals in code at the point where the node is created or bound.

Avoid wiring the same signal in both places. If a scene connection must move into code, remove the matching [connection] entry from the scene in the same change and add a focused contract test for the new owner.

Generated Artifacts

Build outputs and local export artifacts stay out of git. Android exports such as *.apk, *.aab, *.apks, and *.idsig, plus local Python caches such as __pycache__/ and *.pyc, should be produced locally or in release automation, then cleaned from the working tree when no longer needed.

Automated Test Gate

Use res://scripts/tests/run_all_tests.gd as the single headless test entrypoint. It discovers *_test.gd suites under scripts/tests/ and *_probe.gd contracts under scripts/debug/; each discovered file must expose run_all() -> Dictionary with passed, total, failed, and failures.

Local command: godot --headless --path . --script res://scripts/tests/run_all_tests.gd. GitHub Actions runs the same gate and also rejects tracked generated artifacts such as APKs, build output, logs, temp files, Python bytecode, and Android export output.

Pre-commit GDScript Checks

The repo includes .pre-commit-config.yaml, gdformatrc, and gdlintrc for gdtoolkit's gdformat and gdlint. Install Python 3, then run python -m pip install pre-commit and pre-commit install to enable local checks before commit.

GitHub Actions validates the pre-commit config and runs gdformat --check plus gdlint on changed *.gd files. This is intentionally changed-file enforcement while the historical full-repo gdformat/gdlint baseline is being ratcheted down. The legacy combat_controller.gd file is temporarily excluded from gdformat only, because formatter adoption for that file would be a large standalone churn pass; gdlint, script-load validation, tests, and the size roadmap still cover it.

Asset Path Contracts

Runtime asset maps and fallback constants should be validated through res://scripts/tests/asset_pipeline_test.gd. When adding a new stable fallback texture path, expose it from the owning class's asset_contract_paths() helper so the shared runner can prove the import exists and loads.

assets.json is the production inventory, not a runtime texture registry. Keep its schema and res:// references valid, and keep runtime texture paths in resources/visual/first_pass_asset_map.json or resources/art/assetgen/runtime/manifest.json where the validation test can import-load them.

Script Size Budget

Shop, collection, and run-summary scripts have a warning-only size budget checked by res://tools/quality/script_size_budget.gd. Oversized scripts need a documented exception in docs/script_size_budget.html with a concrete removal path.
