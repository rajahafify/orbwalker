Summary

The project has a strong visual identity: ornate gothic framing, premium-feeling orb art, a memorable main menu, and enough cohesion to look like a real game rather than a raw prototype. The current weakness is not art volume. It is readability, hierarchy, screen-to-screen polish consistency, and how clearly screenshots communicate the core loop.

Overall assessment: the game is artistically ahead of its UX and launch-readiness polish.

itch.io Demo / Alpha Page - 7 / 10

The game can plausibly be shown on itch.io soon as an early demo or alpha. The screenshots already communicate a distinctive fantasy match-combat style, and the presentation is stronger than many early itch builds.

Remaining gaps:

  • Clearly label the build as alpha/demo.
  • Improve combat and shop readability for first-time players.
  • Make the first run easy to understand without external explanation.
  • Keep expectations scoped to an in-progress prototype.

Publisher Pitch Vertical Slice - 5 / 10

The project has a strong art foundation and a recognizable genre promise, but the current screenshots still read as prototype-dense. A publisher will likely ask whether players can immediately understand what matters on screen, what the decision space is, and what makes the game commercially distinct.

Remaining gaps:

  • Cleaner combat hierarchy: board, enemy state, player state, and action feedback should read in that order.
  • Stronger first-session onboarding and tutorial clarity.
  • More consistent polish between the main menu, combat, shop, victory, and defeat/results screens.
  • A curated vertical slice that proves the core loop is fun, legible, and repeatable.
  • Pitch materials that show progression, reward, and differentiation beyond the visual style.

Steam Coming Soon Page - 5.5 / 10

The game has enough identity to start preparing Steam store assets, but the current screenshot set is not yet ideal for a public Steam page. Steam screenshots need to survive thumbnail size, quick scrolling, and comparison against polished commercial games.

Remaining gaps:

  • Curate 4-6 screenshots that clearly show the core loop.
  • Improve text contrast and spacing in dense UI areas.
  • Make the board and player choices more visually central in combat screenshots.
  • Bring result and shop screens closer to the premium quality of the main menu.
  • Prepare capsule art and a short trailer before public page launch.

Steam Playable Demo / Next Fest-Style Demo - 4 / 10

The current presentation is not yet enough for a Steam demo beat. Steam demo players need a stable, understandable, replayable slice with strong UX and a clear hook within the first few minutes.

Remaining gaps:

  • First-run tutorialization and decision guidance.
  • Balance confidence across early runs.
  • Clear reward and progression pacing.
  • Strong bug polish and runtime stability.
  • Mouse/touch/controller expectation clarity.
  • Enough content variety to support repeat play.
  • Store-facing build QA and demo-specific acceptance testing.

Highest-Leverage Next Polish

The fastest way to raise all launch-readiness ratings is a focused vertical-slice polish pass:

  1. Combat readability and visual hierarchy.
  2. Shop decision clarity.
  3. Result screen polish to match the main menu quality.
  4. First-run onboarding.
  5. Screenshot composition and store-page asset curation.
  6. Stability and QA on the complete run loop.
